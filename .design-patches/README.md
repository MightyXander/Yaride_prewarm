# Патчи: плавные загрузки и анимации

Дроп-ин патчи под ваш стек (Vite + React 19 + TS, **framer-motion**, inline-стили
+ CSS-переменные). Новых зависимостей нет. Навигация/логика/гейтинг/заглушки —
**не тронуты** (см. NAVIGATION_AND_LOGIC.md). Меняется только оформление загрузок.

## Что чинят (4 причины «рваного» перехода)

1. **Скелетон ≠ реальный экран.** Раньше при загрузке показывались 2 простые
   серые карточки **без Hero**. Hero потом «выскакивал», карточки меняли
   геометрию → контент прыгал. Теперь скелетон — **структурный двойник**:
   `HeroSkeleton` + `TripCardSkeleton` 1:1 по геометрии. Появление данных без сдвига.
2. **`AnimatePresence mode="wait"`** гасил скелетон ПОЛНОСТЬЮ, потом с паузой
   показывал контент → мигание пустотой. Заменено на **`mode="popLayout"`**:
   уходящий скелетон «выпадает» в `absolute` и кроссфейдится с входящим контентом
   на том же месте.
3. **`--muted` не была определена** → плейсхолдеры были фактически прозрачными.
   Добавлена в `.light`/`.dark` + цвета скелетона `--skel-base` / `--skel-sheen`.
4. **Пульсация по opacity** заменена на мягкий **shimmer-блик**.

Всё уважает `prefers-reduced-motion` (shimmer и кроссфейды гаснут, переходы → мгновенные).

## Файлы

| Патч | Куда | Действие |
|------|------|----------|
| `src/index.css.additions.css` | `src/index.css` | вставить блоки (+ удалить старый `@keyframes pulse`) |
| `src/components/ui/Skeleton.tsx` | новый файл | примитивы `Skeleton` и `Slot` |
| `src/components/HeroSkeleton.tsx` | новый файл | двойник `Hero` |
| `src/components/TripCardSkeleton.tsx` | **замена** | структурный двойник `TripCard` |
| `src/screens/MainScreen.tsx` | **замена** | `popLayout`-кроссфейд + twin-скелетон |

`Appear.tsx` менять не нужно — он используется как есть (stagger карточек сохранён).

## Применение

1. Скопируйте новые/заменяемые файлы по путям из таблицы.
2. В `src/index.css` вставьте переменные в `.light` и `.dark`, добавьте
   `.skeleton` / `@keyframes skeleton-shimmer` / reduced-motion, удалите старый
   `@keyframes pulse`.
3. `npm run build` — проверка типов.

## Те же принципы для остальных экранов (по желанию)

Скелетоны на других экранах сейчас на `var(--secondary)` с пульсом — замените на
`Skeleton` тем же приёмом (геометрия = реальная):

- **`PassengerRequestScreen`** — блок «Маршрут» при загрузке: вместо двух
  disabled-`Select` с `placeholder="Загрузка…"` поставьте два
  `<Skeleton h={48} r={16} />` в той же `Card` (высота уже совпадает — CLS нет).
- **`MyTripsScreen`** — блоки на `var(--secondary)` с `animation: pulse` → `Skeleton`.

### «Кроссфейд на месте» (Slot) — высший пилотаж

Если хотите, чтобы текст проявлялся **ровно поверх** своего плейсхолдера (как в
прототипе-эталоне), оборачивайте конкретные значения в `Slot`:

```tsx
import { Slot } from '../components/ui/Skeleton';

<Slot ready={!loading}>{driver.name}</Slot>
<Slot ready={!loading} r={14} barStyle={{ background: 'rgba(0,0,0,.12)' }}>
  <Chip variant="brand">{seats} мест</Chip>
</Slot>
```

Реальный контent задаёт геометрию, маска гаснет при `ready` — нулевой сдвиг.
