## 1. Плавные загрузки (исходная задача) — что и почему

**Симптом:** при загрузке экран «рвался» — скелетон не совпадал с реальной вёрсткой, контент прыгал, мигала пустота.

**4 причины и решения:**
1. **Скелетон ≠ реальный экран.** Раньше при загрузке — 2 простые серые карточки **без Hero**; потом Hero «выскакивал», карточки меняли геометрию. → Скелетон теперь **структурный двойник**: `HeroSkeleton` + `TripCardSkeleton` 1:1 по геометрии. CLS = 0.
2. **`AnimatePresence mode="wait"`** гасил скелетон полностью, затем с паузой показывал контент → мигание пустотой. → **`mode="popLayout"`**: уходящий скелетон «выпадает» в `absolute` и кроссфейдится с входящим контентом на месте.
3. **`--muted` не была определена** (использовалась в скелетоне и disabled-кнопках) → плейсхолдеры были прозрачными. → Добавлены `--muted`, `--skel-base`, `--skel-sheen` в `.light`/`.dark`.
4. **Пульсация по opacity** → заменена на мягкий **shimmer-блик**.

**Доп. принципы:**
- **Кроссфейд «на месте» (0 сдвига):** реальный контент всегда отрисован и задаёт геометрию; поверх лежит shimmer-маска, которая гаснет при готовности — текст проявляется ровно там, где стоял плейсхолдер.
- **Строгий reduced-motion** + **мгновенный fallback к финальному кадру** (`data-revealed` / `splashGone`): на слабых устройствах и в фоновых вкладках, где CSS-переход «не тикает», экран всё равно доходит до данных, без залипания.
- **Профиль грузится синхронно из памяти** → без скелетона (намеренно, как в доке).

## 2. Редизайн карточки поездки, Hero и навбара (итеративно, по запросам владельца)

> Всё это — **косметика/композиция** (разрешено). Перечислено с обоснованием. Для ИИ-ревьюера ключевое — в разделе 3 «Не откатывать».

### Карточка поездки (`TripCard`)
- **Кол-во поездок** перенесено **под аватар** (ниже бейджа рейтинга), формат «иконка-авто + число» **без слова «поездок»** — экономит место, не обрезается. *Причина: в строке с именем счётчик не помещался и обрезался.*
- **Адрес → только улица** (район убран). *Причина: направление уже видно в шапке («Брагино → Центр»), район избыточен.*
- **Строка «модель · цвет»** вместо «модель · цена». *Причина: цена перенесена в кнопку мест (ниже).*
- **Цена — в кнопке количества мест** (правый столбец): «≈100 ₽ / N мест», в две строки. *Причина: явный ценовой акцент у действия/брони.*
- **Чип цены/мест — нейтральный** (был бренд-жёлтый). *Причина: снижение доли жёлтого (см. §2 «Полировка»).*
- **Плашка гос-номера**: субтильная подложка в стиле РФ-номера на **токенах темы** (сливается с карточкой), **без региона/«RUS»** (только серия+номер), **справа на строке «модель · цвет»**. Имя — на всю ширину сверху (длинные имена просто обрезаются многоточием, плашка не ломается). *Причина: компактно, читаемо, и в паре строк.*
- Чип «N мест» — одна строка, с пробелом («3 места»); карточка получила тонкий верхний bevel-хайлайт для глубины.

### Hero
- **Иллюстрация**: была еле видна (opacity .14, в углу). Сначала сделали заметную «бегущую» линию маршрута, затем по запросу — **статичные «радар-кольца»** (мотив локации/«вы здесь»), **без анимации**, заметно, в обеих темах. Плюс мягкое статичное свечение.
- **Заголовок крупнее/жирнее** (17 → 22, вес 800) и **на всю ширину**; CTA «Ближайшая в HH:MM» — **широкой строкой ниже**, рядом ⇄ и (демо) ＋. *Причина: «герой» должен звучать; крупный заголовок ровно в 2 строки.*
- Сохранены: бренд-градиент + анимированная рамка (прод-фича), kicker «Сегодня по маршруту».

### Нижний навбар (`FloatingNav`)
- **Колокол уведомлений перенесён из шапки в навбар**: 3 секции — **слева 🔔 (действие→тост), по центру «Поездки», справа «Профиль»**. Индикатор-пилюля ездит между двумя **табами** (Поездки/Профиль). *Причина: запрос владельца; шапка чище.*

### Splash (первый запуск)
- **Логотип «поехали вместе» + слоган**, ~2.5 c, данные грузятся в фоне → когда сплэш уходит, экран уже готов; **без белой вспышки**.

### Полировка «дороже» (визуальный проход)
- **Меньше жёлтого**: жёлтый — точечный акцент (Hero + аватар-идентичность); чипы нейтрализованы.
- **Тёплый off-white** (светлая тема): `--background`/`--card` сдвинуты из холодного в тёплый.
- **Контраст `muted`-текста в тёмной теме** поднят (`#a3a3ad → #b3b3bd`).
- **Табличные цифры** (`tabular-nums`) для времени/цены.

## 3. ⚠️ Намеренные отклонения — НЕ откатывать как баг (для ИИ-ревьюера)

Эти пункты выглядят как «расхождение с мокапом/доком», но сделаны осознанно и согласованы:

1. **Карточка показывает ЦВЕТ авто и ГОС-НОМЕР; цена — в кнопке мест; адрес — только улица.** Это новая презентация `TripCard`. **Требует бэкенда:** поля `car_color` и `plate` (+ опционально регион). Номер — «если есть» (для авто без номера плашку не показывать). Не «чинить» обратно на «модель · ≈цена ₽» и полный адрес.
2. **Счётчик поездок в карточке — голое число с иконкой** («🚗 213»), без слова. ⚠️ При этом `pluralize(n,…)` **сохранён** для счётчика в Hero («N поездок/поездки/поездок») и в списках — там правило из доки не нарушено. Не возвращать слово в карточку и не убирать pluralize из Hero.
3. **Колокол уведомлений — в навбаре (3 ячейки), как ДЕЙСТВИЕ, а не таб.** Маршрутизация/гейтинг навбара не изменены: активная пилюля по-прежнему между Поездки/Профиль. Колокол — заглушка-тост (как и был в шапке). Не считать «третьим табом».
4. **Кнопка «＋ создать поездку» в прототипе видна всегда** — только для демонстрации вида. В проде **сохраняется гейтинг: ＋ только у водителя** (`userRole === 'driver'`), у пассажира её нет. Не убирать гейтинг при переносе.
5. **Чип мест из бренд-жёлтого стал нейтральным** и **тёплый off-white фон** — это правка цветов **в рамках токенов тем** (разрешено §4/§5). Не возвращать к чисто-белому/жёлтому «как в мокапе».
6. **Hero переструктурирован** (заголовок на всю ширину, CTA строкой ниже) и **иллюстрация другая (кольца, без анимации)** — композиция/иконки/анимации разрешены. Не восстанавливать старую угловую линию-каракулю.

## 4. Что НЕ трогали (сохранено как в `NAVIGATION_AND_LOGIC.md`)

- Модель навигации (конечный автомат `useNavigation`, `navigate`/`goBack`), направленный слайд экранов, резерв под навбар.
- Ролевая модель и **гейтинг ＋** (только водитель); deep-link; направление коридора (load по corridor, не по time_slot).
- Тема только в Профиле (луна/солнце), персист, приоритет ручного выбора.
- Самобронь (нельзя забронировать свою; серая кнопка + тост).
- Намеренные заглушки: верификация телефона в брони (демо «любые 4 цифры»), колокол-уведомления (тост), модерация ВУ.
- Зафиксированные строки копирайта (слоган, «Сегодня по маршруту поездок нет», «Сменить тему» и т.п.).
- Скелетоны в полную высоту (без CLS); профиль синхронно из памяти.
- Стек-ограничения: только inline-стили + `index.css`, без Tailwind/shadcn/новых зависимостей, framer-motion для переходов.
