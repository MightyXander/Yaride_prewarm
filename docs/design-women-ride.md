# Дизайн-док: батч фич «Женская поездка» (Yaride mini-app)

Дизайн внутри существующей дизайн-системы Yaride (React+TS, mobile-first, авто dark-mode,
шрифт Manrope). Никаких новых брендов и палитр — только уже объявленные токены
(`src/index.css`) и готовые компоненты (`src/components/ui/*`, `ThemeModeSheet`).
Код не пишем — это спека для задач на вёрстку.

## Общие принципы этого батча

- **Один смысловой акцент на экран.** Жёлтый бренд (`--brand`) = выбор/подтверждение;
  красный `--gradient-danger` = только SOS (ничего больше в красном).
- **Цвет никогда не единственный сигнал** (WCAG `color-not-only`): серая карточка
  всегда сопровождается текстом-причиной, выбранный пол — галочкой/весом, SOS — иконкой+текстом.
- **Тач-таргеты ≥44px** (SOS ≥60px), анимации 150–260ms на `transform/opacity`,
  `prefers-reduced-motion` уважается (как в `ThemeModeSheet`/`index.css`).
- **Копирайт**: активный залог, sentence case, честность, empty-state = приглашение к действию.
  Нейтральный тон, объясняем «зачем», без запугивания.

### Используемые токены (уже существуют)

| Назначение | Токен |
|---|---|
| Акцент выбора/подтверждения | `--brand` #ffdd2d, `--gradient-brand`, `--brand-foreground` #18170f |
| Тёмный бренд (иконка/галочка) | `--brand-dark` #d4b400 |
| Опасность (только SOS) | `--gradient-danger`, `--danger`, `--danger-foreground`, `--shadow-danger` |
| Успех/подтверждено | `--success`, `--success-foreground` |
| Поверхности | `--card`/`--elevated`, `--surface`, `--accent`, `--secondary` |
| Текст | `--foreground`, `--muted-foreground` |
| Поля ввода | `--field`, `--field-border`, `--field-shadow` |
| Границы/радиусы | `--border`, `--radius` 20, `--radius-xl` 28 |
| Неактивное (см. ниже) | `filter: grayscale(60%); opacity: .6` (конвенция репо) |

### Единственный новый ассет

В `Icons.tsx` добавить один symbol `i-copy` (две смещённые скруглённые рамки,
stroke, viewBox 0 0 24 24) — нужен для кнопки «Скопировать» в safety-share.
Пример пути: `<rect x="9" y="9" width="11" height="11" rx="2"/><path d="M5 15V5a2 2 0 0 1 2-2h10"/>`.
Всё остальное закрывается существующими иконками (`i-shield`, `i-sos`, `i-share`,
`i-user`, `i-check`, `i-pin`, `i-phone`, `i-car`, `i-telegram`).

## Фича 1 — Выбор пола (регистрация + профиль)

### 1.1 Компонент `GenderSelect` (сегмент из 2 чипов)

Новый презентационный компонент на базе `Chip.tsx` (variant `default` уже даёт
`--gradient-brand`+`--brand-foreground` для selected, `--secondary` для невыбранного).
Не dropdown: выбор из 2 значений лучше показать открытым сегментом — 1 тап вместо 2,
оба варианта всегда видны (Progressive disclosure не нужен).

```
Пол                                    (label, sectionLabelStyle-паттерн)
┌───────────────┐ ┌───────────────┐
│    Женский    │ │    Мужской    │    2 чипа, gap 10px, каждый flex:1
└───────────────┘ └───────────────┘
Нужно для режима женских поездок       (hint, 12px muted)
```

**Значения:** `female` → «Женский», `male` → «Мужской» (порядок: женский первым —
фича про женскую безопасность). Значение `unknown` в UI не выбирается, только дефолт БД.

**Токены/размеры:**
- Контейнер: `display:flex; gap:10px`. Каждый чип `flex:1` (перекрывает `minWidth:60`).
- Высота чипа 44px (дефолт `Chip`), `borderRadius:14px`.
- Label: 12px, `text-transform:uppercase`, `letter-spacing:.05em`, `font-weight:700`,
  `color:var(--muted-foreground)`, `margin-bottom:6px` (паттерн `sectionLabelStyle`
  из SafetyScreen:15-22 — переиспользовать точь-в-точь).
- Hint: 12px, `color:var(--muted-foreground)`, `line-height:1.5`, `margin-top:6px`.

**Состояния:**
| Состояние | Вид |
|---|---|
| Не выбрано | оба чипа `--secondary` / `--secondary-foreground`, weight 600 |
| Выбрано | активный чип `--gradient-brand` / `--brand-foreground`, weight 700; второй — secondary |
| Ошибка | под сегментом сообщение `--danger`, 12.5px, weight 600; чипы получают `box-shadow: inset 0 0 0 1.5px var(--danger)` до первого выбора |
| Press | `scale(.94)` (встроено в `Chip`) |

**Копирайт (ru):**
- Label: `Пол`
- Hint (регистрация): `Нужно для режима женских поездок — женщины смогут ехать только с женщинами.`
- Ошибка: `Укажите пол`

**a11y:**
- Обёртка `role="radiogroup"` + `aria-label="Пол"`; каждый чип `role="radio"`
  `aria-checked={selected}`. (Chip рендерит `<button>` — добавить проп прокидывания role/aria.)
- Ошибка в `aria-live="polite"` (role="alert") рядом с группой; фокус — на первый чип
  (`focus-management`).
- Контраст: `--brand-foreground` #18170f на `--brand` = ~11:1 ✓; `--secondary-foreground`
  на `--secondary` ✓ в обеих темах.

## Фича 2 — Женская поездка: разделение списка + серые карточки + empty

Контекст: тумблер `women_only` уже в SafetyScreen (:204-211). Бэк пока **не фильтрует по
`driver_sex`** → логика клиентская в `MainScreen`. Нужен новый проп у `TripCard`:
`disabledReason?: string` (и/или `dimmed?: boolean`).

### 2.1 Поведение когда `women_only = true`

Показываем **ВСЕ** поездки, но двумя секциями с заголовками-разделителями:
1. **Женские** (driver_sex==='female') — активные, обычные карточки, бронируются.
2. **Остальные** (male/unknown) — grayscale+opacity, не бронируются, с пояснением почему.

Когда `women_only = false` — список без секций, как сейчас (обратная совместимость).

```
┌─ Женские поездки ───────────────  3 ─┐   section header
│  [ TripCard активная ]               │
│  [ TripCard активная ]               │
├─ Остальные — с мужчинами ────────────┤   section header (muted)
│  [ TripCard серая, не тыкается ]     │
│  [ TripCard серая, не тыкается ]     │
└──────────────────────────────────────┘
```

### 2.2 Заголовок-разделитель секции (`SectionHeader`)

Лёгкий, не карточка — sticky-подзаголовок в потоке списка.

- Layout: `flex; align-items:center; gap:8px; padding:14px 4px 8px`.
- Текст: 13px, `font-weight:800`, `letter-spacing:.01em`.
  - «Женские поездки» — `color:var(--foreground)`, слева иконка `i-shield` 15px в
    `--brand-dark` (спокойный акцент, не заливка).
  - «Остальные — с мужчинами» — `color:var(--muted-foreground)`, без иконки.
- Счётчик справа: pill `--secondary`/`--secondary-foreground`, 12px weight 700,
  `height:19px; padding:0 7px; border-radius:999px` (как plate-бейдж TripCard:200-219).
- Первый заголовок без верхнего отступа (list padding уже есть).

**Копирайт (ru):**
- `Женские поездки`
- `Остальные — с мужчинами`
- (Счётчики — числом.)

### 2.3 Серая (неактивная) карточка

Переиспользуем `TripCard`, добавляем визуальный слой + мини-пояснение. НЕ новый компонент.

- Обёртка карточки: `filter: grayscale(60%); opacity:.6` (конвенция репо) + `pointer-events`
  оставляем на карточке (можно раскрыть детали), но **кнопка брони заблокирована**
  и заменена пояснением.
- Кнопка «Забронировать» → неактивный вид (`opacity:.5; background:var(--muted);
  color:var(--muted-foreground); cursor:not-allowed`) — точно как booked-состояние
  (TripDetails :364-369). Плюс под именем/адресом — строка-причина.
- **Мини-пояснение** (строка внутри карточки, под моделью авто): `flex; gap:6px;
  align-items:center; font-size:12px; color:var(--muted-foreground); margin-top:4px`,
  иконка `i-user` 12px. Текст: см. копирайт.
- `aria-disabled="true"` на карточке-действии; в `aria-label` добавить причину.
- **Важно:** grayscale — не единственный сигнал (WCAG). Причина текстом обязательна.

**Копирайт мини-пояснения:**
- `Водитель — мужчина. Недоступно в режиме женских поездок.`
  (коротко в карточке; если тесно — `Водитель-мужчина — вне женского режима`)

### 2.4 Empty-состояние (женских нет)

Когда `women_only=true` и в секции «Женские» 0 поездок. Компактный честный блок-приглашение
(в духе `EmptyTripsState`, но со своим текстом). Секция «Остальные» при этом всё равно
показывается ниже — уехать можно.

```
        ◐  (i-shield 40px в кружке --secondary)
   По вашему маршруту женских поездок
   сейчас нет
   Можно уехать с мужчинами — SOS и
   «поделиться с близкими» работают в
   любой поездке.
   [ Настроить маршрут ]   ← вторичная, если применимо
```

- Контейнер: `Card` или центрированный блок, `padding:24px 16px`, `text-align:center`
  (единственное место с центрированием — уместно для empty).
- Иконка: 40px `i-shield` в кружке 56px `--secondary`, `color:var(--muted-foreground)`.
- Заголовок: 15px weight 700 `--foreground`; текст: 13px `--muted-foreground` `line-height:1.5`,
  `max-width:34ch` (читаемая длина строки).
- Отступ до секции «Остальные» — 16px.

**Копирайт (ru), финальный (адаптация формулировки основателя, короче):**
- Заголовок: `По вашему маршруту женских поездок сейчас нет`
- Тело: `Можно уехать с мужчинами — SOS и «Поделиться с близкими» доступны в любой поездке.`
- (Опц. кнопка вторичная): `Изменить маршрут`

**Заметки по вёрстке:**
- Группировку делать в `MainScreen` перед рендером списка: разбить массив на
  `female` / `rest` по `driver_sex`; рендерить `SectionHeader` + список.
- `TripListItem` (`types/api.ts`) расширить `driver_sex`. Пока бэк не отдаёт —
  фильтр клиентский; когда отдаст, тот же UI.
- Порядок внутри секций — как сейчас (по времени; `isNext`-подсветка только в «Женских»).
- Анимация появления секций — stagger как у существующего `Appear`/`AppearList` (не менять).

## Фича 4 — «Поделиться с близкими» (safety-share)

Новая кнопка + bottom-sheet. **Отличается от существующего вирального «Поделиться»**
(`i-share`, зовёт попутчиков через `shareToTelegram`, TripDetails:382): safety-share отдаёт
доверенному человеку ПОЛНУЮ инфу о поездке для контроля безопасности.

### 4.1 Точка входа (кнопка)

В блоке действий поездки (`actionButtons`, TripDetails:380-443), рядом с существующим
«Поделиться», но семантически иначе:

- Существующее «Поделиться» (виральный инвайт) — оставить как есть (`variant="ghost"`,
  `i-share`), но переименовать в **`Позвать попутчиков`** (снять коллизию смысла).
- Новая кнопка **`Поделиться с близкими`** — `variant="secondary"`, иконка `i-shield`
  (безопасность, не соц-граф), minHeight 48px. Открывает sheet.

**Копирайт кнопок:**
- Виральная (была «Поделиться»): `Позвать попутчиков`
- Safety-share (новая): `Поделиться с близкими`

260ms `cubic-bezier(.25,.1,.25,1)`, закрытие по фону/Esc, drag-handle сверху,

Формируется рядом с `handleShareTrip` (TripDetails:292-300), но полнее (водитель, авто,
номер, дата) — использовать поля `trip.driver`, `trip.car`, `trip.plate`, `trip.time`,
`trip.route`, `buildTripDeepLink`.

### 4.4 Кнопки и flow

| Кнопка | Вид | Действие |
|---|---|---|
| `Отправить {имя}` | `primary` (`--gradient-brand`), minHeight 48 | если доверенный контакт задан: `shareToTelegram(text, link)` (или tel/sms-намерение на contact.phone). Подпись подставляет имя контакта |
| `Скопировать` | `secondary`, flex:1, иконка `i-copy` | `navigator.clipboard.writeText(text+link)` → success-состояние |
| `Поделиться…` | `ghost`, иконка `i-share` | системный `navigator.share({title,text,url})`; фолбэк `shareToTelegram` вне поддержки (сейчас в `share.ts` navigator.share ОТСУТСТВУЕТ — добавить) |

**Состояния:**
| Состояние | Вид/копирайт |
|---|---|
| Idle | все кнопки активны (или см. «нет контакта») |
| Скопировано | кнопка «Скопировать» → `--success`/`--success-foreground`, иконка `i-check`, текст `Скопировано`; авто-возврат через 2s (`success-feedback`, `toast-dismiss`) |
| Нет доверенного контакта | вместо primary — карточка-подсказка `--accent`: текст + кнопка `Добавить контакт` → ведёт в SafetyScreen (блок «Доверенный контакт»). «Скопировать»/«Поделиться…» остаются доступны |
| navigator.share недоступен | кнопка «Поделиться…» скрыта или фолбэк на Telegram-шаринг (без ошибки) |
| Ошибка clipboard | toast `Не удалось скопировать` |

**Копирайт (ru):**
- Заголовок: `Поделиться с близкими`
- Подзаголовок: `Отправьте детали поездки тому, кому доверяете.`
- Лейблы сводки: `Маршрут` / `Когда` / `Водитель` / `Авто` / `Номер`
- Primary с контактом: `Отправить {имя}`
- Primary без контакта (карточка): заголовок `Нет доверенного контакта`,
  тело `Добавьте близкого человека — сможете отправлять ему поездки в один тап.`,
  кнопка `Добавить контакт`
- `Скопировать` → после: `Скопировано`
- `Поделиться…`
- Toast: `Не удалось скопировать`

**a11y:**
- Наследует диалоговую семантику `ThemeModeSheet` (`role=dialog`, `aria-modal`, Esc,
  фокус-трап желателен: при открытии фокус на заголовок/первую кнопку, `modal-escape`).
- Кнопки-иконки с текстом — ок; чисто-иконочных нет.
- «Скопировано» — `aria-live="polite"` (success-фидбэк озвучивается).
- Сводка поездки — обычный текст (читается скринридером), не только визуал.
- Контраст сводки: value `--foreground`, label `--muted-foreground` на `--secondary` — AA ✓.

## Чек-лист ui-ux-pro-max §1–3

### §1 Accessibility (CRITICAL)
- [x] Контраст ≥4.5:1: бренд-фон/текст ~11:1; SOS белый на danger ≥3:1 для large-bold ✓;
      muted-текст на своих поверхностях проверен в обеих темах.
- [x] Цвет не единственный сигнал: серая карточка + текст-причина; выбранный пол —
      вес+галочка; SOS — иконка+текст; armed — обводка+текст, не только пульс.
- [x] Aria: `radiogroup/radio` (пол), `dialog/aria-modal` (шиты), `aria-live` на SOS-armed
      (assertive) и «Скопировано» (polite), `aria-disabled` на серых карточках.
- [x] Focus: видимое `.focus-ring` на всех интерактивных; фокус на первую ошибку/первую кнопку шита.
- [x] Reduced-motion: пульс SOS и слайд шита отключаются (глобальный медиазапрос).

### §2 Touch & Interaction (CRITICAL)
- [x] Таргеты: чипы пола 44px, кнопки 48px, SOS 64px, строки шита 48-52px — всё ≥44.
- [x] Отступы между таргетами ≥8-10px (ряды кнопок gap 8-10).
- [x] Press-фидбэк: `pressable` (`scale .97`), `Chip` (`scale .94`), haptic на SOS (warning/error),
      на выбор темы (light) — переиспользуем `haptics.ts`.
- [x] Загрузка/сохранение: оптимистичный апдейт + toast-откат (как `persistToggle`); disabled
      на кнопках во время сохранения контакта.
- [x] Не полагаемся на hover: всё работает по тапу.

### §3 Performance (HIGH)
- [x] Без layout-shift: armed-обводка и серый слой через `box-shadow`/`filter`/`opacity`
      (transform/opacity/filter), не border/width → нет reflow (`layout-shift-avoid`).
- [x] Анимации 150–260ms на compositor-свойствах; `Slot`-кроссфейд уже гасит мигание
      «дефолты→данные» (SafetyScreen:182-185) — тот же приём для нового блока пола.
- [x] Секции списка используют существующий `Appear`/stagger — без доп. тяжёлого рендера;
      группировка — один проход `map`/reduce в `MainScreen`.
- [x] Один новый ассет (`i-copy`, инлайн-SVG) — без сетевых запросов/растровых картинок.
